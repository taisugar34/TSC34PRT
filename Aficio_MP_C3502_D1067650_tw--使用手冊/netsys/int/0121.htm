snmp

使用「snmp」命令，顯示和編輯社群名稱等SNMP組態設定。

檢視設定

只有安裝NetWare時，才會顯示「IPX」。

msh> snmp

  • 預設的存取設定1如下所示：

    社群名稱：public

    IPv4位址：0.0.0.0

    IPv6位址：：：

    IPX位址：00000000:000000000000

    存取種類：唯讀

    有效協定：IPv4／IPv6／IPX

  • 預設的存取設定2如下所示：

    社群名稱：admin

    IPv4位址：0.0.0.0

    IPv6位址：：：

    IPX位址：00000000:000000000000

    存取種類：讀寫

    有效協定：IPv4／IPv6／IPX

  • 如果加上「-p」，則可以逐一地檢視設定。

  • 若要顯示目前的社群，請指定其登錄編號。

顯示

msh> snmp ?

社群名稱設定

msh> snmp "number" name "community_name"

  • 可以設定編號為從1-10的十個SNMP存取設定。

  • 如果沒有在1-10號中登記「public」，就無法從SmartDeviceMonitor for Admin或SmartDeviceMonitor for Client存取印表機。變更社群名稱時，請使用SmartDeviceMonitor for Admin和SNMP設定工具來與印表機設定對應。

  • 社群名稱最多可輸入15個字元。

刪除社群名稱

msh> snmp "number" clear name

存取種類設定

msh> snmp "number" type "access_type"

協定設定

使用下列命令，設定協定為「有效」或「無效」。如果將設定協定為「無效」，則停用該協定的所有存取設定。

只有安裝NetWare時，才會顯示「IPX」。

msh> snmp {ipv4|ipv6|ipx} {on|off}

  • 為IPv4指定「ipv4」，為IPv6指定「ipv6」，為IPX／SPX指定「ipx」。

  • {on}表示「有效」，而{off}表示「無效」。

  • 無法同時關閉所有協定。

每個登錄編號的協定的設定

只有安裝NetWare時，才會顯示「IPX」。

msh> snmp "number" active {ipv4|ipv6|ipx} {on|off}

  • 若要變更存取設定的協定，請使用下列命令。但是如果使用以上命令停用了協定，則在此處啟動該協定將無效。

存取設定

只有安裝NetWare時，才會顯示「IPX」。

msh> snmp "number" {ipv4|ipv6|ipx} "address"

  • 可以根據所使用的協定來設定主機位址。

  • 網路介面卡只接受來自位址為IPv4、IPv6和IPX，及存取種類為「唯讀」或「讀寫」的主機的請求。輸入「0」使網路介面卡接受來自任何主機的請求，而不要求特定的存取種類。

  • 輸入主機位址來傳送「trap」存取種類資訊。

  • 若要指定IPv4或IPv6，輸入「ipv4」或「ipv6」後接一個空格，然後再輸入IPv4或IPv6位址。

  • 若要指定IPX／SPX，輸入「ipx」後接一個空格，IPX位址後面接一位小數，然後輸入網路介面卡的MAC位址。

sysLocation設定

msh> snmp location "character string"

刪除sysLocation

msh> snmp clear location

sysContact設定

msh> snmp contact

刪除sysContact

msh> snmp clear contact

SNMP v1v2功能設定

msh> snmp v1v2 {on|off}

  • 指定「on」以啟用，或指定「off」以停用。

SNMP v3功能設定

msh> snmp v3 {on|off}

  • 指定「on」以啟用，或指定「off」以停用。

SNMP TRAP設定

msh> snmp trap{v1|v2|v3} {on|off}

  • 指定「on」以啟用，或指定「off」以停用。

遠端設定授權設定

msh> snmp remote {on|off}

  • 指定「on」以啟用，或指定「off」以停用SNMP v1v2設定。

SNMP v3 TRAP設定顯示

msh> snmp v3trap

msh> snmp v3trap [1-5]

  • 如果輸入一個從1到5的編號，則只顯示該編號的設定。

為SNMP v3 TRAP設定傳送位址

只有安裝NetWare時，才會顯示「IPX」。

msh> snmp v3trap [1-5] {ipv4|ipv6|ipx} "address"

為SNMP v3 TRAP設定傳送協定

只有安裝NetWare時，才會顯示「IPX」。

msh> snmp v3trap [1-5] active {ipv4|ipv6|ipx} {on|off}

為SNMP v3 TRAP設定使用者帳號

msh> snmp v3trap [1-5] account "account_name"

  • 帳號名稱最多可輸入32個文數字字元。

刪除SNMP v3 TRAP使用者帳號

msh> snmp v3trap [1-5] clear account

設定SNMP v3加密演算法

msh> snmp v3auth {md5|sha1}

設定SNMP v3加密

msh> snmp v3priv {auto|on}

  • 將自動加密組態設定設為「auto」。

  • 如果選擇「on」，純文字通訊變為無效－只能進行加密通訊。

  • 「加密的通訊」表示機器上已設定加密的密碼。

  • 若要使用SNMP設定工具，您必須先從製造商網站下載SmartDeviceMonitor for Admin並安裝至電腦。關於下載SmartDeviceMonitor for Admin的詳細資訊，請聯絡您的當地經銷商。
