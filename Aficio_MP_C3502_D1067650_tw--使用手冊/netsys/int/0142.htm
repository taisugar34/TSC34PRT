系統記錄資訊

使用下列指令來顯示系統記錄資訊：

  • UNIX：使用「syslog」參數，及「rsh」、「rcp」、「ftp」、「sftp」指令。"無法在Windows環境下，使用「rsh」與「rcp」。

  • telnet：使用「syslog」指令。

本機在IKE階段1中的ISAKMP SA<Hash演算法1>不符合通訊主機的ISAKMP SA<Hash演算法2>。請確定本機的ISAKMP SA的Hash演算法與通訊主機的相符。

SMTPC: failed to get smtp server ip-address.

無法獲取SMTP伺服器IP位址。可能的原因有：

  • 找不到指定的DNS伺服器。

  • 沒有建立與網路的連線。

  • 找不到指定的DNS伺服器。

  • 指定了不正確的DNS伺服器。

  • 在DNS伺服器中找不到指定的SMTP伺服器IP位址。

SMTPC: failed to connect smtp server. timeout.

由於逾時，與SMTP伺服器的連線失敗。這可能是由於指定的SMTP伺服器名稱不正確、未建立網路連線，或網路設定不正確，所以沒有來自SMTP伺服器的回應。查看SMTP伺服器名稱，或網路連線和設定。

SMTPC: refused connect by smtp server.

與SMTP伺服器的連線被拒絕。這可能是由於指定了SMTP伺服器以外的其他伺服器，或指定的SMTP伺服器連接埠編號不正確。查看SMTP伺服器名稱、連接埠編號或SMTP伺服器連接埠編號。

SMTPC: no smtp server. connection close.

由於沒有來自SMTP的回應，與SMTP伺服器的連線失敗。這可能是由於指定SMTP伺服器以外的其他伺服器，或指定的SMTP伺服器連接埠編號不正確。查看SMTP伺服器名稱、連接埠編號或SMTP伺服器連接埠編號。

SMTPC: failed to connect smtp server.

與SMTP伺服器的連線失敗。這可能是由於未建立網路連線、或網路設定不正確，因此沒有來自SMTP伺服器的回應，或指定的SMTP伺服器名稱不正確、或在DNS伺服器中找不到指定的SMTP伺服器IP位址、或指定了SMTP伺服器以外的其他伺服器，或指定的SMTP伺服器連接埠編號不正確。請查看DNS伺服器的IP位址和SMTP伺服器的IP位址、或SMTP伺服器名稱和SMTP連接埠編號、或SMTP伺服器的SMTP連接埠編號，或網路連線和設定。

SMTPC: username or password wasn't correct. [回應代碼]（資訊）

由於指定的SMTP使用者名稱不正確、或指定的SMTP密碼不正確，導致到SMTP伺服器的連線失敗。查看SMTP使用者名稱和密碼。
